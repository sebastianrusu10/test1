Domain 1

Foundational concepts and terminology.

Core Concepts

The Internet

  • The Internet: The world's largest public network.
  • Intranet: A website that is internal to a business and not seen by the public.
  • Extranet: If you have an intranet and decide to allow some members from outside inside, you have an extranet.

Types of Networks

Peer-to-Peer

  • Most home and small office networks.
  • No central device managing and storing users, files, and folders.
  • Every device is its own server because every device is responsible for managing access to its own resources.

Client-Server

  • Network present in many medium to large-sized businesses where a central server hosts all the user accounts, devices, and permissions for the users and devices.
  • Active Directory: This is the server piece in a Windows-based client-server network.

Data Transmission Types

  1. Unicast: Data is transmitted from 1 device to another.
  2. Multicast: One device transmits data to a selected group of devices.
  3. Broadcast: A device transmits data to every device on its network.

IoT & Virtualization

IoT & Virtualization

  • IoT (Internet of Things): Device that can be controlled remotely.
  • Hypervisor: Tool that manages physical resources for a VM. Inside hypervisors, we have VMs.
  • Virtual Machines (VMs): Software-based fully functional instances of an operating system.
  • Type 2 Hypervisors: Like Type 1 hypervisors, Type 2 also manage and allocate system resources. They are located within operating systems.

Virtual Switches

  • External: Gives the virtual machine using the switch access to whatever the host PC has access to.
  • Internal: Will have access to themselves and the host.
  • Private: Will only have access to each other.

VPN & Remote Access

VPN (Virtual Private Network)

  • Virtual Private Network (VPN): A private network that uses a public one to transmit data from a source to destination.

Domain 2

LANs, addressing, topologies, wireless concepts, and WAN / internet access technologies.

LAN & Addressing

LAN (Local Area Networks)

  • LAN (Local Area Networks): Networks that are generally confined to a single building or an area of a building. They share common resources.

Wired LAN

  • Computers and other devices use Ethernet cables to connect to switches.
  • The switches connect through Ethernet cables to one or more routers which connect to a network or other networks.

Wireless LAN

  • Devices connect to each other through a Wireless Access Point often called a Wireless router.

IP Addressing & DMZ

APIPA (Automatic Private IP Addressing)

  • Range: 169.254.x.x
  • Used when a device cannot get an IP address through a DHCP server.

Private IP Addresses

  • A private IP address is not routable on a public network.
  • Ranges:
  • 10...
  • 192.168...
  • 172.16 - 172.31
  • (127... is loopback)

DMZ (Demilitarized Zone)

  • A type of LAN Perimeter network.
  • Holds devices that need to be seen by both the public and private security zones of a network.
  • Back to Back config: Firewall on both sides of a perimeter network.
  • Three-leg perimeter: (Diagram shows Public connecting to Firewall, which splits to Private and Perimeter network).

Topologies

Network Topologies

Star

  • A common topology for small networks is a star topology.
  • A central device (usually a switch) is the centerpiece.
  • All devices connect to it.

Bus

  • Every device on a network is connected to a backbone cable that runs from one end to the other.
  • Terminators at both ends of the cable.
  • As the signal travels from one device to another, it can bounce off the end if needed.
  • Without terminators, signals bounce and the network won't function.
  • Best for small networks.
  • Any change can bring the entire network down.

Ring

  • Mostly used in WAN.
  • Popular 20 years ago, less common now.
  • Fiber Distributed Data Interface (FDDI) networks.
  • Fiber optic cable network in a ring.
  • Data transmission is token-based.
  • A break in the ring takes the network down.
  • FDDI has 2 rings.

Mesh

  • Full mesh connections: h(h - 1) / 2 (h = number of devices)
  • Common in WAN.
  • Every device has a connection to every other device.

Repeaters

  • Wireless bridges can also serve as repeaters.
  • A signal that is reaching maximum distance can be boosted.

Wireless

Wireless Networks

  • Wireless networks operate on IEEE 802.11.
  • 2.4 GHz vs 5 GHz
  • 5 GHz: faster, shorter range.
  • The 2 frequencies are not compatible on the same network.
  • 802.11n supports both.

Types of Network Security

  • WEP (Wired Equivalent Privacy): Weak, has been cracked.
  • WPA (Wi-Fi Protected Access): Stronger.
  • WPA2: Strongest (802.11i).

Port-based network access control

  • Can be used with wired or wireless.
  • Supplicant Auth Network.

Two types of wireless networks

  1. Infrastructure
  2. P2P (Point to Point / Ad hoc)

Infrastructure vs. P2P

  • Infrastructure: One in which one or more wireless access points are the central point of the network.
  • The Service Set Identifier (SSID) is the name of the network.
  • P2P: Has no central wireless access point.
  • 2 devices connect to each other.
  • Not as secure.
  • Wireless bridging: Used to connect two wired segments of a network.
  • In bridge mode, networks with different 802.11 standards can connect to each other.

WAN & Internet Access

WAN (Wide Area Network)

  • Networks that cover multiple geographic areas and are usually a collection of LANs.

Dial-up Line

  • Still the only means of getting to the outside world for some users.
  • A phone line and a server that accepts dial-up connections are needed.
  • A very, very slow connection by today's standards.
  • The top advertised speed is 56 Kbps.
  • 38 Kbps or so is about as fast as it gets.

Leased Lines

  • Dedicated connection from one location to another.
  • Whoever leases the line has the full consistent bandwidth of that line.

T-Carrier Lines

  • T1: Uses 24 64 Kbps channels of data plus another 8 for overhead for a total of 1.544 Mbps.
  • Customers can lease a partial T1 line.
  • T3: 44.736 Mbps.
  • Equivalent to 28 x T1 lines.
  • Uses 672 64 Kbps channels.
  • Can lease partially.

ISDN (Integrated Services Digital Network)

2 Types

  1. Basic Rate Interface (BRI): Uses 2 64 Kbps channels for a total speed of 128 Kbps (more than double that of a dial-up).
  2. Primary Rate Interface (PRI): It uses 23 64 Kbps channels for a total speed of 1.536 Mbps. Runs on a T1 circuit line.

Internet Connections (DSL, Cable, Cellular, Satellite)

DSL (Digital Subscriber Line)

  • Digital Subscriber Line: A customer has a dedicated phone line from the origin to a telco office.
  • Synchronous DSL (SDSL): Upload and download speeds are the same. Often used in businesses that need fast uploads.
  • Asynchronous DSL (ADSL): Most commonly used in homes and businesses. Different upload and download speeds (download speed being higher).

Cable Modems

  • Speeds tend to be faster overall than DSL.
  • DSL connections are on dedicated lines and cable modem connections are typically not.
  • Cable internet tends to run over shared bandwidth.
  • Speeds tend to be slower during peak usage hours.

Cellular & Satellite

  • 5G - highest
  • 4G LTE is faster than 4G
  • Speed varies based on proximity to a tower
  • Satellite

Digital Lines (E1/T1)

  • E1 and E3: European equivalent.
  • E1: 32 64kbps channels = 2.048 Mbps.
  • E3: 16 x E1 lines = 34.368 Mbps.
  • Slowest to fastest: T1, E1, E3, T3.

VPN Security (IPsec)

IPsec (Internet Protocol Security)

  • Many VPNs use IPsec.
  • Protocols that authenticate and encrypt data packets over VPN.
  • Provides confidentiality and integrity of data which is crucial for making sure data over VPN stays private.

IPsec Protocols

  1. Security Association (SA): Generates the auth and encryption keys used in IPsec.
  2. Auth Header (AH): Provides auth and integrity of data but not data encryption.
  3. Encapsulating Security Payload (ESP): Provides auth, integrity, and encryption of data.

Domain 3

Switching, routing, transmission media, and performance/QoS.

Switching (Layer 2)

Network Devices: Hubs vs. Switches

Hubs (Legacy/Physical Layer)

  • Function: Hubs are unintelligent devices that do not store MAC addresses. They simply receive a signal and broadcast (flood) it out to all connected ports.
  • Efficiency: Very inefficient; connected devices must check every packet to see if it is meant for them.
  • Current Use (Repeaters): Hubs are largely extinct for directing traffic but are useful as repeaters.
  • Signal Boosting: Standard Ethernet cables (Twisted Pair) have a max distance of 328 feet (100 meters). A hub placed in the middle can boost the signal to extend this range (e.g., covering a 500ft run).
  • Topology: Directly connected routes on a hub usually involve just 2 devices.

Switches (Data Link Layer/Layer 2)

  • Function: Intelligent devices that use MAC Address Tables to direct traffic only to the specific destination device.
  • MAC Learning:
    When a device connects and transmits data, the switch learns its MAC address (hardware address).
    If the switch knows the destination MAC, it sends data directly to that port.
    If the switch doesn't know the destination MAC, it floods the data to all ports to find the recipient.
  • Maintenance: Entries in the MAC table are erased from memory after a period of inactivity to save space.

Switch Architecture & Features

Switching Methods

  1. Store and Forward: The switch receives the entire frame and checks for errors (CRC check) before sending.
    Pros: No corrupt data sent.
    Cons: Slower (higher latency).
  2. Cut-Through: The switch reads only the header (to find the destination MAC) and forwards immediately.
    Pros: Faster.
    Cons: Does not check for errors; corrupt frames may be propagated (discarded at destination).

Port Types & Cables

  • Standard Ports: Typically 10 Mbps, 100 Mbps (Fast Ethernet), 1 Gbps (Gigabit), or 10 Gbps.
  • Uplink Ports / MDI: Faster ports designed to connect switches together.
  • Straight-through cables: Used for connecting unlike devices (Switch <-> Router) or when using Uplink ports.
  • Crossover cables: Historically used to connect like devices (Switch <-> Switch).
  • Note: Most modern switches handle straight/crossover automatically (Auto-MDIX).

Performance Specs

  • Backplane Speed: The total throughput a switch is capable of handling at any one time.
  • Math Example: A 48-port switch running at 100 Mbps per port has a potential throughput of 4.8 Gbps (48 x 100 Mbps).

Types of Switches

  • Unmanaged: Plug-and-play; no configuration possible.
  • Managed: Configurable by administrators. Supports IP addresses, VLANs, and security settings.
  • Layer 2 Switch: Uses MAC addresses only; cannot route between different networks.
  • Layer 3 (Multilayer) Switch: Can use IP addresses to route traffic between networks (inter-VLAN routing).

Switch Security & Redundancy

Security Options

  • Port Security: Managed switches can memorize specific MAC addresses allowed on a port.
  • Example: If a port is configured to allow only 1 specific address, it prevents rogue devices from plugging in and accessing the network.

Redundancy & Spanning Tree Protocol (STP)

  • Redundancy: Setting up multiple links/devices so if one fails, another takes over (failover). It can also assist with load balancing (sharing workload to increase performance).
  • The Problem (Loops): Redundant links can accidentally create physical loops (e.g., plugging a switch into itself). This causes Broadcast Storms, where broadcast frames flood the switch endlessly, crashing the network.
  • The Solution (STP):
    Spanning Tree Protocol (STP) uses the Spanning Tree Algorithm (STA).
    It transmits BPDUs (Bridge Protocol Data Units) to detect loops.
    It automatically blocks (destroys) redundant paths to prevent loops while keeping a backup path available.

Switches, Ports, and VLANs

Indicators & Ports

  • On a switch, a yellow light indicator usually indicates that a connection is running at less than full speed.
  • For example, a Fast Ethernet cable plugged into a Gigabit Ethernet port will run at Fast Ethernet speed and display a yellow light.
  • Uplink ports tend to be faster than regular ports. They also accept both straight-through and crossover cables.

Unmanaged Switch

  • An unmanaged switch has no configuration to set up. Thus, it is also known as a plug-and-play switch.

VLAN Assignment

  • If more users are assigned to Project Management, which uses VLAN2, the floor plan will adjust so more physical ports are assigned from VLAN1 to VLAN2.

Port Security & Control

  • Administrators can use switches to control the flow of network traffic.
  • They do this by controlling the number of MAC addresses each switch port will memorize.
  • For example, if a port memorizes only one address, then it will deny access to any rogue devices that do not use that MAC address.

Redundancy & Load Balancing

  • Hardware redundancy can also have the benefit of load balancing, which allows multiple devices to share a workload.

Backplane Speed

  • The number of ports X the port speed = backplane speed.
  • 100 Mbps x 48 ports = 4.8 Gbps.
  • Thus 4.8 Gbps is the backplane speed for this switch.

Forwarding Methods

  • The data transmission method for a switch that receives a frame, checks it for errors, and then sends it is the store and forward method.
  • In the cut-through method of data transmission, once a destination MAC address is known, the frame is sent.

Hubs vs. Switches

  • Hubs do not store any intelligent information. Switches store MAC addresses in MAC address tables.

Virtual Switches

  • There are three common types of virtual switches: external, which allows for a connection to an external network; internal, which allows for a virtual machine to connect to a host; and private, which allows for virtual networks to be isolated.

Spanning Tree Protocol (STP)

  • A Spanning Tree Protocol (STP) transmits Bridge Protocol Data Units (BPDUs) to identify redundant links in switches.

Routing (Layer 3)

Routers & Routing (Layer 3)

Function

  • Routers direct traffic between different networks using IP addresses.
  • Network Segmentation: Breaking a network physically (different buildings) or logically (VLANs) to improve performance and security.

Routing Tables

  • Stored in the router's RAM.
  • Convergence: The process of routers updating their tables to reflect network changes (topology changes). This can cause temporary slowdowns.
  • Memory Issues: Protocols like BGP (Border Gateway Protocol) create massive tables that can overload memory; RIP uses very little.

Routing Protocols

  1. RIP (Routing Information Protocol):
    Type: Distance-vector.
    Metric: Hop count (number of routers to destination).
    Downside: Does not consider connection speed (a slow path with fewer hops is chosen over a fast path with more hops).
  2. OSPF (Open Shortest Path First):
    Type: Link-state.
    Metric: Uses bandwidth/speed and hops.
    Upside: Smarter routing decisions based on link performance.
  3. Static Routes: Manually defined by an administrator (commands: route add, route delete). Used when traffic must go through a specific router.
  4. Default Route (0.0.0.0): The "Gateway of Last Resort." If traffic is destined for a network the router doesn't know, it is sent here (usually the ISP connection).

Metrics (Priority)

  • The lower the metric, the better the route.
  • Priority Order: Directly Connected Routes (Lowest Metric) < Static Routes < Dynamic Routes.

Routing Concepts and Commands

Directly Connected Routes

  • A directly connected route involves a connection between two devices.
  • Usually, this connection is between a device and a router or two routers.
  • A direct connection is best suited for two routers connected to each other.

Static Route Syntax

  • The syntax for a static route involves the route add command, the network address, the word, MASK, the subnet mask, and then the IP address of the device to route through.
  • Thus, the complete command is: route add 192.168.2.0 MASK 255.255.255.0 192.168.2.1.
  1. Step 1: Type: route add 192.168.2.0 MASK 255.255.255.0 192.168.2.1
  2. Step 2: Select the Enter key on the keyboard

Routing Protocols

  • Transmission Control Protocol/Internet Protocol (TCP/IP) is the default dynamic routing protocol.
  • Routing Information Protocol (RIP) is a distance-vector routing protocol that uses hop counts to determine the best path in which traffic should flow.
  • It would determine that the best path in the image would be Router A to Router D at 10 Mbps, because even though the speed is slower, the route only takes one hop.
  • Open Shortest Path First (OSPF) is a link-state routing protocol that uses hops, speed, and other factors in determining the best routes for a network.
  • Among the options, Border Gateway Protocol (BGP) takes up the most memory within a router because it tends to build very large routing tables.

Routes, Metrics, and Default Route

  • The IP address 0.0.0.0 in a routing table indicates the default route for traffic on that network.
  • The lower the metric (as indicated in the metric column), the better the route. In the image, the route 0.0.0.0 has a metric number of 25, the lowest among the four options provided.
  • A static route is a route used to specify a consistent path from a source to a destination.
  • A dynamic route adjusts how traffic is routed from a source to a destination based on workload, bandwidth, and similar factors.
  • A default route is a route used when no other specific route is listed for a destination IP address or network.

Convergence

  • In convergence, a root bridge router learns the topology of an entire network and then distributes it to the remaining routers on the network.

Transmission Media

Transmission Media (Cables)

Twisted Pair (Copper)

  • Standard: Ethernet, uses RJ-45 connectors (larger than phone RJ-11).
  • Distance Limit: 100 meters (328 feet).
Types
  • UTP (Unshielded): Standard, cheap, twists prevent some EMI.
  • STP (Shielded): Expensive, extra shielding for high EMI areas.
  • Plenum: Fire-retardant jacket, low smoke (for air ducts/ceilings).
Categories
  • CAT3: 10 Mbps (Obsolete).
  • CAT5: 100 Mbps (Fast Ethernet).
  • CAT5e: 1 Gbps (Gigabit).
  • CAT6: 10 Gbps (up to 55 meters).
  • CAT6a: 10 Gbps (full 100 meters).

Fiber Optic

  • Transmission: Uses light (photons) instead of electricity.
  • Pros: Immune to EMI (electromagnetic interference), long distance (up to 70km), high speed (10 Gbps+).
  • Cons: Expensive, strict bend radius.
Types
  • Single-mode: One ray of light, extremely long distances.
  • Multimode: Multiple rays, shorter range (~600m), used for high-speed LAN backbones.

Performance & QoS

Performance & Interference Factors

Interference Sources

  • Wired: Power cables, Fluorescent lighting, EMI, Crosstalk (signal bleeding between wires), damaged cables.
  • Wireless: Heavy machinery (elevators), Physical obstructions (Walls - especially blocks 5GHz), 2.4GHz interference (Microwaves, cordless phones).

Optimization Tools

  • QoS (Quality of Service): Controls how resources are used to prioritize specific traffic (e.g., Voice over IP).
  • Port Forwarding: Configuring a router to send specific traffic to a specific device/port within the LAN.

Security Note

  • Data should be encrypted. Intercepted data is useless without the decryption key.
  • Power fluctuations (spikes/brownouts) can corrupt data transmission.

QoS, VLAN Scenarios, and Media Limits

QoS Setup

  • Through QoS, users can set up rules that designate the priority of app or service traffic.
  • They can select a service, such as Skype, and assign it a priority, such as Bulk, Standard, or Maximum. Maximum indicates the highest priority.
  • They can then save the changes and enable the QoS rule so it takes effect.
  1. Step 1: Select the Service Name drop-down arrow
  2. Step 2: Select skype [ 0 ~ 0 ]
  3. Step 3: Select Add
  4. Step 4: Select the Priority drop-down arrow
  5. Step 5: Select Maximum
  6. Step 6: Select Save

VLAN Scenario

  • The scenario describes three separate groups: students, administrative staff, and a repair shop.
  • The groups are isolated from one another and require different network permissions.
  • From a logical standpoint, three VLANs or logical networks are appropriate for the scenario, one for each group.

Media Limits

  • The top distance for data transmission over twisted-pair cables is 328 feet (100 meters).
  • A multimode fiber optic cable can carry multiple rays of light but the maximum distance the signals can be carried is 600 meters.
  • As multimode fiber optic cables carry signals for a maximum of 600 meters, longer distances need single-mode fiber.
  • Plenum cable is required for running cables through walls and ceilings as plenum cable does not give off a noxious odor in case of fire.

Wireless & Wired Interference

  • For 2.4 GHz networks, channels 1, 6, and 11 are safe to use to avoid overlapping on a 2.4 GHz network.
  • Transmission in wired networks can be affected by electromagnetic interference (EMI), fluorescent lighting, and nearby power cables.
  • Care should be taken to place cables away from these sources.
  • Walls, elevator shafts, and cordless phones (which run on a 2.4 GHz frequency) can all interfere with a wireless network signal.

Network Address Translation or NAT

  • Allows devices with private IP addresses to use public IP addresses to communicate with other networks
  • Private IP addresses are not routable, meaning that data from these devices will not route to other networks, namely the internet
  • Until NAT appeared on the scene
  • The most common form of NAT is Port Address
  • Translation (PAT)
PAT
  • The most common form of NAT is Port Address
  • Translation (PAT)
  • Any data request from a private IP address is assigned a port number
  • The data request is then translated to the public IP
  • address this device has and is routed where it needs to go When data is returned, this device knows the port number for the return data and then translates the address back to the private IP address of the original recipient and sends the data to that address With over 65,000 port numbers available, this literally allows up to 65,000 devices with private IP addresses to share one public IP address for routing purposes
Static NAT
  • A one-to-one mapping between a private IP address and a public IP address
  • For this to work, there needs to be a public IP address available for every device with a private IP address For this reason, it is not commonly used in networks
Dynamic NAT
  • A many-to-many mapping between private IP addresses and public IP addresses to where a pool of public IP addresses is used, and when a device with a private IP address needs to use a public IP address, it does, so long as there is a public IP address available
  • When the connection is done, the public IP address goes back into the pool, waiting to be used again
  • PAT is by far the most common form of NAT used, and it is set up by default on many routers, especially small office/home office routers

Troubleshooting is a 7 step process

Step one: Identify the problem
In this, we probably need to ask open-ended questions to try to figure out why a network connection is not working
Is the problem isolated to one user?
Is an entire building affected?
Networking
A problem cannot be solved unless we know what the problem is

Step two: Establish a theory of probable cause This is a fancy way of saying, find out what caused the problem
Did something get unplugged?
Is there a misconfiguration?
Was a rogue device added to the network to cause a network attack?

Step three: Test the theory
Can we duplicate the problem?
If we can, it might make it easier to find a solution

Step four: Establish a plan to solve the problem
Is this an easy fix?
Are we going to need to reboot a server or do something else that will disrupt services temporarily?

In order to solve a problem, we need to have an idea as to how to solve it

Step five: Implement the solution or escalate it as necessary
Here, we need to understand a very important point: If you are the one trying to solve a problem and get stuck as to how to do it, escalate it
Most network problems are very time-sensitive, and taking too much time to fix something when someone else can do it faster can cause major problems in timing, data loss, lost revenue, and frustrated employees

Step six: Verify that the solution worked
Before declaring to a customer that a problem is solved, test out the solution
Make sure that the original problem cannot be replicated

Step seven: Document the solution
Many companies use some kind of help desk ticketing software, SharePoint site, or even a database to track problems and solutions
Make sure a solution is detailed enough so that someone else, should they encounter the same problem, has an idea of what to do

A tool that is used to find faults in fiber-optic cables, specifically kinks and breaks in the cable

Here is how it works
A signal is sent down the cable
If the signal goes through, the cable can handle a signal just fine If there is a break, the signal will bounce off the break and back to its origin
